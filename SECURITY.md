# Security Policy

## Reporting a vulnerability

Please report security issues privately. Two acceptable channels:

1. **Preferred:** open a [private security advisory](https://github.com/sb-dev/pactwright/security/advisories/new) on this repository.
2. Email the maintainer at `samir.benzenine@gmail.com` with subject line `pactwright security`.

Please include:
- A description of the issue and its impact.
- Steps to reproduce, or a minimal proof of concept.
- The commit SHA or release tag affected.
- Your disclosure timeline expectations.

## Supported versions

During the Phase 1–7 bootstrap, only `main` is supported. Tagged releases will define their own supported windows once the release loop (SPEC.md §21) is in place.

## Response

Best-effort acknowledgement within 7 days. This is a personal-time project — there is no commercial SLA. Critical issues will be prioritised; lower-severity reports may take longer.

## Scope

In scope: code, workflows, and spec-graph tooling in this repository.
Out of scope: GitHub itself, Anthropic's services, and any third-party dependency vulnerabilities that should be reported upstream.
