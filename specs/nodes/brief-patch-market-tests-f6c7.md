## Pinned decisions (the grafts/fixes THIS lane carries)

- **Graft A — one shared predicate, proven by a parity test.** The gate and the two structural rules must count "live competitor" *identically* via the single shared helper. This lane carries the **Fix-3 PARITY TEST** that fails if they ever diverge (the unbypassable guard on B's main con). This lane does not write the helper — it pins its single-definition behaviour from both call sites.
- **Graft A — gate core is a pure `evaluatePatchGate(input)`.** All gate cases (a)-(e) are direct calls on synthetic `{graph, resolvedBrief, today}` inputs — no git, no GitHub, no `$GATE_BASE`. (The git adapter `runPatchGate` belongs to domain-backend and is not unit-tested here.)
- **Graft C — dedicated `patch-comparison` waiver, real expiry.** Case (d) tests `waives → patch-comparison` (never `pr-evidence`), unexpired passes and expired does not, via the `toDateString`/`expires >= today` semantics.
- **Fix 2 — `selected` excluded from the live-competitor set.** A chosen winner (`status: selected`) is NOT a live competitor; the post-selection steady state is clean. This lane carries the Fix-2 test on both the gate and the structural handler.
- **Fix 3 — graph-first PR→brief mapping, fail-closed.** The mapping is graph-first (`patch.branch == head → competes-for` brief), branch-name parse is the fallback, and an unmapped multi-patch merge fails CLOSED while never stranding unrelated PRs. This lane carries case (c): graph mapping vs branch fallback vs no-match-fail-closed.
- **No selection of a winner, no scope widening.** This lane only verifies B's intended behaviour. If, while writing tests, a test cannot be satisfied because intended behaviour appears to have shifted, STOP and return to human approval (CLAUDE.md rule 5) rather than re-scoping the gate.
